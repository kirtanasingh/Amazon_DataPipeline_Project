## Data Pipeline & Preprocessing

### Overview

This project demonstrates the implementation of a reusable and scalable **data preprocessing pipeline** for the **Amazon Fine Food Reviews Dataset**. The pipeline follows industry-standard data engineering and machine learning practices to transform raw data into a clean, structured, and model-ready format.

The preprocessing workflow includes data cleaning, handling missing values and duplicates, feature engineering, train-validation-test splitting, feature scaling, encoding of categorical variables, and modular pipeline creation using Scikit-Learn. The objective is to build a preprocessing system that is reusable, reproducible, and suitable for deployment in real-world machine learning applications.

---

## Aim

To build a reusable and scalable data preprocessing pipeline for the **Amazon Fine Food Reviews Dataset** using industry-standard practices including data cleaning, missing value handling, feature engineering, train-validation-test splitting, and modular pipeline design.

---

## Dataset

**Dataset Name:** Amazon Fine Food Reviews Dataset

**Description:**
The Amazon Fine Food Reviews dataset contains customer reviews of food products sold on Amazon. It includes review text, ratings, product information, user information, and timestamps, making it suitable for sentiment analysis, recommendation systems, and machine learning applications.

---

## Why Data Preprocessing?

Real-world datasets are rarely clean and often contain:

* Missing values
* Duplicate records
* Incorrect data formats
* Unstructured text
* Numerical and categorical variables
* Outliers and inconsistent entries

A preprocessing pipeline ensures:

* Data consistency
* Reproducibility
* Scalability
* Better model performance
* Easy deployment into production environments

## Project Workflow

1. Load the Amazon Fine Food Reviews dataset.
2. Explore the dataset structure and identify data types.
3. Handle missing values.
4. Remove duplicate records.
5. Convert data into appropriate formats.
6. Perform feature engineering where required.
7. Separate numerical and categorical features.
8. Build preprocessing pipelines using Scikit-Learn.
9. Apply scaling and encoding.
10. Split the dataset into Train, Validation, and Test sets.
11. Save the preprocessing pipeline using Joblib for future use.
